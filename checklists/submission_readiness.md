# Submission Readiness Checklist

## Before submission

- [ ] Re-run related work search and update citations.
- [ ] Verify all arXiv IDs and author names.
- [ ] Freeze threat model.
- [ ] Freeze benchmark definitions.
- [ ] Freeze policy version used for experiments.
- [ ] Run all baselines.
- [ ] Run all ablations.
- [ ] Run adaptive attacks.
- [ ] Validate audit replay.
- [ ] Clean artifact.
- [ ] Remove real secrets or private data.
- [ ] Have at least two internal reviewers read the paper.

## Claim audit

For every strong claim in the abstract/introduction:

- [ ] Is there a theorem, experiment, or explicit assumption supporting it?
- [ ] Is the claim limited to runtime-observable tool execution?
- [ ] Does it avoid implying the model cannot be influenced?
- [ ] Does it avoid overstating user confirmation guarantees?

## Figures and tables

- [ ] Architecture figure.
- [ ] Attack examples figure.
- [ ] Label/effect table.
- [ ] Main ASR table.
- [ ] Utility table.
- [ ] Ablation table.
- [ ] Performance table.
- [ ] Related work table.
