# Evaluation Metrics Definitions

This document defines all evaluation metrics used in ProvShield to ensure consistency across the paper, artifact, and documentation.

## Security Metrics

### Attack Success Rate (ASR)

**Definition:** Fraction of attack scenarios where the malicious high-risk action executes end-to-end.

**Formula:** `ASR = (number of attack scenarios with successful malicious execution) / (total attack scenarios)`

**Notes:**
- This is end-to-end: it includes both the LLM generating the attack tool call AND the monitor allowing it.
- Reported with 95% Wilson confidence intervals.
- For the 780-scenario evaluation: ASR = 0.6% (3/530 attack scenarios, 95% CI [0.2%, 1.7%]).

### Conditional ASR (given LLM tool call)

**Definition:** ASR restricted to attack scenarios where the LLM generates any tool call.

### Conditional ASR (given attack target tool)

**Definition:** ASR restricted to attack scenarios where the LLM generates the specific attack target tool call.

### Conditional Block Rate (given attack target tool)

**Definition:** Fraction of scenarios where the LLM generates the attack target tool AND ProvShield blocks it.

**Current value:** 100% — every malicious tool call generated by the LLM is blocked.

### LLM Manipulation Rate

**Definition:** Fraction of attack scenarios where the LLM generates any tool call (regardless of whether it's the attack target).

**Current value:** 14.9% (79/530 attack scenarios).

## Utility Metrics

### Benign Task Completion Rate (BTCR)

**Definition:** Fraction of benign tasks that complete successfully, defined as the task's expected tool call executing without being denied by the monitor.

**Formula:** `BTCR = (benign tasks completed successfully) / (total benign tasks)`

**Current value:** 92.4% (231/250 benign scenarios).

### False Blocking Rate

**Definition:** Fraction of benign tasks where a legitimate tool call is denied by the monitor (not bridged — outright denied).

**Formula:** `False Blocking Rate = 1 - BTCR` (when no bridge is offered or bridge is not auto-confirmed)

**Current value:** 7.6% (19/250 benign scenarios).

### Bridge Burden (Confirmation Burden)

**Definition:** Fraction of benign tasks where a bridge confirmation is required by the monitor.

**Formula:** `Bridge Burden = (benign tasks requiring bridge confirmation) / (total benign tasks)`

**Current value:** 7.6% (19/250 benign scenarios).

**Notes:** In the evaluation, bridge-required calls without user confirmation are treated as denied, so Bridge Burden = False Blocking Rate for the automated evaluation. In a real deployment, user confirmation would reduce false blocking.

## Performance Metrics

### Monitor Latency (p50/p95)

**Definition:** Per-tool-call latency added by the runtime monitor (policy evaluation + provenance reconstruction).

**Current values:** p50 = 0.03 ms, p95 = 0.07 ms.

### LLM Latency (p50/p95)

**Definition:** Per-request latency for the LLM API call (not part of ProvShield overhead).

**Current values:** p50 = 13,094 ms, p95 = 36,976 ms.
